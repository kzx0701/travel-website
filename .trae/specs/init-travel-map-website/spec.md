# 旅行城市点亮地图网站 Spec

## Why
用户希望有一个个人化的旅行/出差/工作学习记录工具，通过点亮中国地图上到访过的城市，直观展示自己的足迹。相比纯文字记录，地图点亮形式更具仪式感和可视化效果，同时支持多用户使用并云端同步数据。

## What Changes
- 初始化 Vue3 项目（Vite + TypeScript + Pinia + Vue Router + Element Plus + TailwindCSS）
- 集成地图组件，实现**三级下钻**（全国 → 省级 → 市级），过渡平滑
  > **更新（2026-07）**：地图引擎已从 ECharts 迁移至 **MapLibre GL JS v5+**，支持全球地球仪 + 无缝下钻。详见 `.trae/specs/map-redesign-globe/spec.md`。
- 集成 Supabase 作为后端（Auth 鉴权 + PostgreSQL 数据库 + RLS 数据隔离）
- 实现邮箱密码注册登录（不强制邮箱确认）与登录态管理
- 实现**地图页核心布局**：顶部导航栏 + 左信息区 + 中地图 + 右信息区
- 实现居住地设置（区县级，独立设置页），居住地所在城市不可点亮
- 实现城市搜索（含港澳台，下钻到市级）与地图点击点亮
- 实现到达记录管理（单日期或日期范围、预设+自定义出行目的、单行短文本备注）
- 实现**多城市行程**关联（记录为主、行程为辅的分组模型）
- 实现统计子页面（统计卡片 + 目的分布饼图 + 地图缩略图）
- 实现时间线子页面（按日期倒序 + 按月分组）
- 实现设置子页面（居住地 / 账号 / 偏好）
- 实现个人主页子页面（点亮地图展示 + 统计概览 + 行程列表，默认私密可公开）
- **暂不实现图片上传功能**（后续迭代）
- **仅亮色模式**（暗色模式后续迭代）
- **桌面端优先**（移动端仅保证可用）
- **一期不做忘记密码功能**

## Impact
- 这是一个全新项目，无现有代码影响
- 关键文件/模块：
  - 项目脚手架与目录结构（严格组件化，杜绝冗长代码）
  - 三级地图组件（全国/省/市下钻，MapLibre GL JS + GeoJSON，支持全球地球仪）
  - Supabase 客户端封装与 RLS 策略
  - 认证模块（注册/登录/路由守卫）
  - 城市数据集（含港澳台，省/市/区县/经纬度）
  - 城市搜索组件（防抖 + 模糊匹配 + 拼音）
  - 地图页布局组件（顶栏 + 左侧 + 中地图 + 右侧）
  - 居住地管理（区县级）
  - 到达记录 CRUD + 行程分组
  - 统计仪表盘
  - 时间线视图
  - 个人主页

## ADDED Requirements

### Requirement: 项目脚手架与架构规范
系统 SHALL 基于 Vue3 + Vite + TypeScript 初始化项目，集成 Pinia、Vue Router、Element Plus、TailwindCSS、ECharts，并配置 ESLint + Prettier。代码 SHALL 严格遵循组件化拆分，单个组件文件避免冗长，通用 UI 与业务逻辑分离，可复用组件抽取到独立目录。

#### Scenario: 项目可运行
- **WHEN** 开发者执行 `npm install && npm run dev`
- **THEN** 项目启动并显示首页框架

#### Scenario: 组件化规范
- **WHEN** 开发者新增功能
- **THEN** 页面按"布局组件 + 业务组件 + 通用组件"分层，单文件控制在合理行数内，复用逻辑抽取为 composable

### Requirement: 三级地图下钻
系统 SHALL 提供中国地图三级下钻能力：全球地球仪（默认，可旋转）→ 全国地图 → 点击省份区域过渡到省级地图 → 点击城市区域过渡到市级地图。过渡 SHALL 平滑（flyTo 动画 + zoom 驱动的图层渐变，避免突兀闪烁）。顶部 SHALL 显示层级面包屑（中国 / XX省 / XX市），可点击回上级。

> **技术实现**：基于 MapLibre GL JS v5+ 的单一渲染器架构，通过 flyTo 相机动画 + zoom 驱动的 layer opacity 插值实现无缝过渡。详见 `.trae/specs/map-redesign-globe/spec.md`。

#### Scenario: 默认进入全球地球仪
- **WHEN** 用户进入地图页
- **THEN** 默认显示可旋转的 3D 全球地球仪，聚焦于中国

#### Scenario: 从地球仪进入中国
- **WHEN** 用户双击中国区域 或 点击面包屑"中国"
- **THEN** 平滑 flyTo 到中国视角，世界国家图层淡出，中国省份图层淡入

#### Scenario: 下钻到省级
- **WHEN** 用户点击某省份区域
- **THEN** 平滑 flyTo 到该省份，城市区域边界渐变淡入，已点亮城市以区域填充高亮，圆点渐变淡出

#### Scenario: 下钻到市级
- **WHEN** 用户在省级地图点击某城市区域
- **THEN** 平滑 flyTo 到该城市，选中城市以蓝色高亮填充

#### Scenario: 市级地图不可点亮
- **WHEN** 用户在市级地图查看
- **THEN** 仅展示区县行政区划边界，区县不可点亮记录

#### Scenario: 面包屑返回
- **WHEN** 用户点击面包屑中的"中国"或"XX省"
- **THEN** 平滑 flyTo 回对应层级，图层 opacity 由 zoom 自动回退

### Requirement: 地图点亮粒度与样式
系统 SHALL 只在省级地图点亮城市，城市点亮后所属省份在全国地图自动汇总点亮。全国地图 SHALL 同时显示省份色块汇总与已点亮城市点位。已点亮城市 SHALL 根据到达次数显示不同点亮强度（颜色深浅分级）。居住地城市 SHALL 以特殊样式标识且不可点亮。

#### Scenario: 全国地图展示
- **WHEN** 用户查看全国地图
- **THEN** 显示省份色块（按该省已点亮城市数/比例加深）+ 已点亮城市点位

#### Scenario: 省级地图点亮城市
- **WHEN** 用户在省级地图点击未点亮城市并提交到达记录
- **THEN** 该城市变为已点亮状态，所属省份在全国地图自动汇总点亮

#### Scenario: 点亮强度分级
- **WHEN** 城市被多次到达
- **THEN** 点亮强度（颜色深浅）根据到达次数递增

#### Scenario: 居住地不可点亮
- **WHEN** 用户点击居住地所在城市
- **THEN** 提示该城市为居住地，不可点亮

### Requirement: 地图页核心布局
系统 SHALL 在地图页采用"顶部导航栏 + 左信息区 + 中地图 + 右信息区"四区布局。

#### Scenario: 顶部导航栏
- **WHEN** 用户在地图页
- **THEN** 顶部显示页面切换（地图/统计/时间线/个人主页/设置）+ 用户信息（头像/邮箱/居住地/退出登录）

#### Scenario: 左信息区
- **WHEN** 用户在地图页
- **THEN** 左侧显示城市搜索框 + 已点亮城市列表（可切换排序：最近到达/到达次数/拼音）+ 快捷统计（已点亮城市数/覆盖省份数/总出行次数）

#### Scenario: 右信息区默认态
- **WHEN** 用户未选中城市
- **THEN** 右侧显示层级面包屑 + 操作引导提示（如"点击城市查看详情"）

#### Scenario: 右信息区城市详情态
- **WHEN** 用户选中某城市
- **THEN** 右侧显示城市名 + 所属省份 + 到达次数 + 到达记录列表 + 添加记录按钮

#### Scenario: 右信息区记录表单态
- **WHEN** 用户点击添加/编辑记录
- **THEN** 右侧展开记录表单（不弹窗），含日期/目的/备注字段

### Requirement: 城市搜索
系统 SHALL 提供城市搜索功能，支持按名称/拼音模糊匹配中国城市（含港澳台）。搜索结果 SHALL 显示城市名 + 省份。选中搜索结果 SHALL 下钻到该城市市级地图并展开城市详情。

#### Scenario: 搜索城市
- **WHEN** 用户输入城市名称或拼音关键字
- **THEN** 显示匹配的城市列表（防抖处理）

#### Scenario: 选中搜索结果
- **WHEN** 用户点击搜索结果中的城市
- **THEN** 地图下钻到该城市市级地图，右侧展开城市详情

### Requirement: 居住地设置
系统 SHALL 允许用户在独立设置页设置和修改居住地（精确到区县级）。居住地所在城市 SHALL 不可点亮，在地图上以特殊样式标识。修改居住地后，旧居住地城市变为可点亮，新居住地城市变为不可点亮。

#### Scenario: 设置居住地
- **WHEN** 用户在设置页选择区县作为居住地
- **THEN** 系统保存居住地信息，该居住地所在城市在地图上以特殊样式标识且不可点亮

#### Scenario: 修改居住地
- **WHEN** 用户更改居住地
- **THEN** 旧居住地所在城市变为可点亮状态，新居住地所在城市变为不可点亮状态

#### Scenario: 未设置居住地
- **WHEN** 用户尚未设置居住地
- **THEN** 所有城市均可点亮，并提示用户设置居住地

### Requirement: 到达记录管理
系统 SHALL 允许用户对每个已点亮城市添加多次到达记录。每条记录 SHALL 包含：到达日期（单日期或日期范围）、出行目的（预设分类+用户自定义）、文字备注（单行短文本，上限 50 字）。城市总到达次数为该城市所有记录数。添加记录入口有三：地图点击城市、右侧常驻"添加记录"按钮、左侧城市列表项"+"按钮。

#### Scenario: 添加到达记录
- **WHEN** 用户对已点亮城市添加新记录
- **THEN** 保存记录并更新城市总到达次数

#### Scenario: 首次点亮城市
- **WHEN** 用户对未点亮城市添加第一条到达记录
- **THEN** 城市变为已点亮状态

#### Scenario: 查看城市记录
- **WHEN** 用户点击已点亮城市
- **THEN** 右侧显示该城市所有到达记录列表

#### Scenario: 编辑记录
- **WHEN** 用户编辑某条到达记录
- **THEN** 更新记录内容

#### Scenario: 删除记录
- **WHEN** 用户删除到达记录
- **THEN** 从记录列表中移除

#### Scenario: 删除最后一条记录
- **WHEN** 用户删除某城市的最后一条到达记录
- **THEN** 城市变为未点亮状态

#### Scenario: 日期范围记录
- **WHEN** 用户添加日期范围记录
- **THEN** 系统记录起止日期，时间线与统计按范围处理

### Requirement: 多城市行程关联
系统 SHALL 采用"记录为主、行程为辅"的数据模型：到达记录为主表，可选关联到同一行程（trip_id 分组）。一个行程可关联多个城市的记录，便于一次出行多城市场景。

#### Scenario: 创建行程
- **WHEN** 用户添加记录时选择归属到新行程
- **THEN** 创建行程并关联该记录

#### Scenario: 行程关联多城市
- **WHEN** 用户在一次出行中添加多个城市记录并归属同一行程
- **THEN** 这些记录通过 trip_id 关联，可在个人主页行程列表中分组展示

#### Scenario: 独立记录
- **WHEN** 用户添加记录时不归属行程
- **THEN** 记录独立存在，trip_id 为空

### Requirement: 出行目的分类
系统 SHALL 预设出行目的分类（旅游/出差/探亲/工作学习/中转/其他），并允许用户自定义新增分类。用户自定义分类 SHALL 归属当前用户（RLS 隔离）。

#### Scenario: 选择预设目的
- **WHEN** 用户添加到达记录
- **THEN** 可从预设分类中选择出行目的

#### Scenario: 新增自定义目的
- **WHEN** 用户在目的选择中新增分类
- **THEN** 保存为该用户私有分类，后续可选

### Requirement: 用户认证
系统 SHALL 提供邮箱密码注册和登录功能，基于 Supabase Auth 实现。注册后不强制邮箱确认即可登录。支持退出登录。一期不提供忘记密码功能。

#### Scenario: 注册成功
- **WHEN** 用户提交有效邮箱和密码
- **THEN** 创建账户并可立即登录

#### Scenario: 登录成功
- **WHEN** 已注册用户输入正确凭证
- **THEN** 跳转到地图页并保持登录态

#### Scenario: 未登录访问受保护页面
- **WHEN** 未登录用户访问受保护页面
- **THEN** 重定向到登录页

#### Scenario: 退出登录
- **WHEN** 用户点击退出登录
- **THEN** 清除登录态并跳转到登录页

### Requirement: 数据隔离
系统 SHALL 基于 Supabase RLS（Row Level Security）确保每个用户只能查看和操作自己的数据（居住地、点亮城市、到达记录、行程、自定义出行目的分类）。

#### Scenario: 数据隔离
- **WHEN** 用户 A 登录
- **THEN** 只能看到用户 A 自己的所有数据

### Requirement: 统计子页面
系统 SHALL 提供统计子页面，展示用户的出行数据概览，包括已点亮城市数、覆盖省份数、总出行次数、出行目的分布饼图、点亮地图缩略图。

#### Scenario: 查看统计
- **WHEN** 用户进入统计页
- **THEN** 显示统计卡片 + 目的分布饼图 + 点亮地图缩略图

#### Scenario: 无数据状态
- **WHEN** 用户尚未点亮任何城市
- **THEN** 显示空状态提示

### Requirement: 时间线子页面
系统 SHALL 提供时间线子页面，按时间倒序展示用户所有到达记录，支持按月份分组展示。时间线条目包含城市、日期、目的、备注。

#### Scenario: 查看时间线
- **WHEN** 用户进入时间线页
- **THEN** 按日期倒序展示所有到达记录

#### Scenario: 按月分组
- **WHEN** 时间线展示多条记录
- **THEN** 按月份分组展示

### Requirement: 设置子页面
系统 SHALL 提供独立设置子页面，包含居住地设置（区县级）、账号信息、偏好设置。

#### Scenario: 修改居住地
- **WHEN** 用户在设置页修改居住地
- **THEN** 保存新居住地并更新地图标识

### Requirement: 个人主页子页面
系统 SHALL 提供个人主页子页面，展示点亮地图（只读）、统计概览、行程列表。默认私密，用户可选择公开生成链接。

#### Scenario: 查看自己的主页
- **WHEN** 用户进入个人主页
- **THEN** 展示点亮地图 + 统计概览 + 行程列表

#### Scenario: 公开主页
- **WHEN** 用户开启公开
- **THEN** 生成分享链接，他人可通过链接查看（只读）

#### Scenario: 私密状态
- **WHEN** 主页为私密
- **THEN** 他人无法访问

### Requirement: UI 与视觉
系统 SHALL 使用 Element Plus 作为 UI 组件库，TailwindCSS 作为原子化样式补充，仅亮色模式。整体视觉风格为"净背景 + 突出点亮"。配色规范：
- 页面背景：纯白（#FFFFFF）
- 已点亮城市主色：暖橙系（如 #FF6B35）
- 点亮强度三档：1 次浅、2-3 次中、4+ 次深（具体色值在设计稿中确定）
- 居住地城市标识色：冷蓝系（如 #3B82F6）
- 未点亮城市点位/省份色块：淡灰（如 #E2E8F0）
- 卡片/按钮圆角与阴影、字体方案：待设计稿确认后落地

#### Scenario: 组件库统一
- **WHEN** 开发者使用 UI 组件
- **THEN** 优先使用 Element Plus，自定义样式用 TailwindCSS

#### Scenario: 桌面优先
- **WHEN** 用户在桌面浏览器访问
- **THEN** 提供完整四区布局体验；移动端仅保证基本可用，不做深度优化

#### Scenario: 配色落地
- **WHEN** 地图渲染点亮状态
- **THEN** 已点亮城市用暖橙色（按次数三档深浅），居住地用冷蓝色，未点亮用淡灰，背景纯白

### Requirement: 城市数据集
系统 SHALL 内置中国城市数据集（含港澳台），覆盖省/市/区县层级与经纬度，用于地图渲染与搜索。GeoJSON 数据 SHALL 打包到项目内（离线可用）。

#### Scenario: 数据覆盖
- **WHEN** 用户搜索或点击地图
- **THEN** 可定位到含港澳台的所有省/市/区县

### Requirement: 数据加载策略
系统 SHALL 采用按需加载策略：进入地图页时加载用户概览数据（已点亮城市列表、统计概览），城市详情与到达记录在选中城市时按需加载，避免一次性加载全部数据。

#### Scenario: 概览数据加载
- **WHEN** 用户进入地图页
- **THEN** 加载已点亮城市列表与统计概览，地图渲染点亮状态

#### Scenario: 城市记录按需加载
- **WHEN** 用户选中某城市
- **THEN** 按需加载该城市的到达记录列表

### Requirement: 地图交互
系统 SHALL 支持鼠标滚轮缩放、拖动平移、缩放按钮（+/-）三种地图交互方式。

#### Scenario: 缩放与平移
- **WHEN** 用户在地图上滚轮缩放或拖动平移
- **THEN** 地图响应交互，并提供 +/- 缩放按钮辅助

### Requirement: 头像生成
系统 SHALL 使用随机卡通头像（如 DiceBear）为用户生成头像，显示在顶部导航栏与个人主页，无需用户上传。

#### Scenario: 头像展示
- **WHEN** 用户登录后查看顶部导航栏
- **THEN** 显示基于随机种子生成的卡通头像

### Requirement: 删除二次确认
系统 SHALL 对所有删除操作（到达记录、行程、居住地、自定义分类等）执行二次确认，避免误删。

#### Scenario: 删除确认
- **WHEN** 用户触发任何删除操作
- **THEN** 弹出确认框，用户确认后才执行删除

### Requirement: 密码强度
系统 SHALL 要求注册密码最少 6 位，无复杂度强制要求，优先体验。

#### Scenario: 密码校验
- **WHEN** 用户注册时输入密码
- **THEN** 校验密码长度不少于 6 位

### Requirement: 左侧城市列表筛选
系统 SHALL 在左侧已点亮城市列表提供按省份、按出行目的、按时间范围三种筛选能力，与排序切换并存。

#### Scenario: 筛选城市列表
- **WHEN** 用户选择筛选条件
- **THEN** 列表按条件过滤展示

### Requirement: 统计页时间筛选
系统 SHALL 在统计页提供时间范围筛选（全部 / 今年 / 最近一年 / 自定义时间范围），统计数据随筛选动态更新。

#### Scenario: 时间筛选统计
- **WHEN** 用户选择时间范围
- **THEN** 统计卡片与图表按该时间范围重新计算

### Requirement: 时间线页筛选
系统 SHALL 在时间线页提供按城市、按出行目的、按时间范围三种筛选能力。

#### Scenario: 筛选时间线
- **WHEN** 用户选择筛选条件
- **THEN** 时间线按条件过滤展示

### Requirement: 个人主页公开访问
系统 SHALL 通过随机 token URL 实现个人主页公开访问。用户开启公开后生成随机 token 链接，他人可通过链接只读访问。可随时关闭公开或重新生成 token 使旧链接失效。

#### Scenario: 生成公开链接
- **WHEN** 用户开启公开
- **THEN** 生成含随机 token 的链接

#### Scenario: 失效旧链接
- **WHEN** 用户重新生成 token 或关闭公开
- **THEN** 旧链接失效，他人无法访问

### Requirement: 行程列表展示
系统 SHALL 在个人主页与统计页以卡片列表形式展示行程，每张卡片含行程名、日期、含城市数、记录数，点击卡片展开详情查看关联的城市记录。

#### Scenario: 查看行程列表
- **WHEN** 用户进入个人主页或统计页
- **THEN** 以卡片列表展示行程，点击可展开详情

### Requirement: 右侧记录列表排序
系统 SHALL 在右侧城市详情中的到达记录列表按到达日期倒序展示。

#### Scenario: 记录排序
- **WHEN** 用户查看城市记录列表
- **THEN** 按到达日期倒序展示

### Requirement: 行程删除处理
系统 SHALL 在删除行程时让用户选择：保留关联记录（trip_id 置空）或同时删除关联记录。选择后执行对应操作并二次确认。

#### Scenario: 删除行程保留记录
- **WHEN** 用户选择保留记录并确认删除行程
- **THEN** 行程删除，关联记录的 trip_id 置空，记录保留

#### Scenario: 删除行程同步删除记录
- **WHEN** 用户选择同步删除并确认
- **THEN** 行程及其关联记录全部删除

### Requirement: 自定义分类删除处理
系统 SHALL 在删除用户自定义出行目的分类时，将使用该分类的已有记录自动转为"其他"分类。

#### Scenario: 删除分类迁移记录
- **WHEN** 用户删除某自定义分类
- **THEN** 使用该分类的记录 purpose 字段转为"其他"，分类删除

### Requirement: 加载状态展示
系统 SHALL 使用骨架屏（skeleton）展示数据加载状态，避免内容闪烁带来的视觉突兀。骨架屏 SHALL 贴合内容区域形态（地图、列表、卡片等）。

#### Scenario: 加载展示
- **WHEN** 数据正在加载
- **THEN** 对应区域显示骨架屏，加载完成后平滑替换为真实内容

### Requirement: 多设备登录
系统 SHALL 采用 Supabase 默认登录行为，不限制多设备同时登录，数据变更通过 Supabase 实时或重新拉取同步。

#### Scenario: 多设备登录
- **WHEN** 用户在多设备登录
- **THEN** 均可正常访问，不互踢
